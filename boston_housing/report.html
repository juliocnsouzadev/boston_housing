Implementação: Definir uma Métrica de Desempenho¶

difcil medir a qualidade de um modelo dado sem quantificar seu desempenho durante o treinamento e teste. Isso geralmente feito utilizando algum tipo de mtrica de desempenho, atravs do clculo de algum tipo de erro, qualidade de ajuste, ou qualquer outra medida til. Para este projeto, voc ir calcular o coeficiente de determinao, R2, para quantificar o desempenho do seu modelo. O coeficiente da determinao para um modelo uma estatstica til em anlise regressa, como se ele frequentemente descrevesse como "good" a capacidade do modelo de fazer estimativas.

Os valores para R2 tm um alcance de 0 a 1, que captura a porcentagem da correlao ao quadrado entre a estimativa e o valor atual da varivel alvo. Um modelo R2 de valor 0 sempre falha ao estimar a varivel alvo, enquanto que um modelo R2 de valor 1, estima perfeitamente a varivel alvo. Qualquer valor entre 0 e 1 indica qual a porcentagem da varivel alvo, ao utilizar esse modelo, ele pode ser explicado pelos atributos. Um modelo pode dar tambm um R2 negativo, que indica que o modelo no melhor do que aquele que estima ingenuamente a mdia da varivel alvo.

Para a funo performance_metric na clula de cdigo abaixo, voc ir precisar implementar o seguinte:

  • Utilizar o r2_score do sklearn.metrics para executar um clculo de desempenho entre y_true e y_predict.
  • Atribuir a pontuao do desempenho para a varivel score.

from sklearn.metrics import r2_score

def performance_metric(y_true, y_predict):
    """ Calcular e retornar a pontuao de desempenho entre 
        valores reais e estimados baseado na mtrica escolhida. """
    score = r2_score(y_true, y_predict)
    # Devolver a pontuao
    return score

Questão 7 - Busca em Matriz¶

O que a tcnica de busca em matriz (grid search) e como ela pode ser aplicada para otimizar um algoritmo de aprendizagem?

Resposta:
A escolha de um modelo consiste em basicamente 3 passos:

  1. Treinar vrios modelos (com hiperparmetro diferentes) com os dados de treinamento
  2. Usar os dados de validao para escolher com base em um score (F1 score por exemplo) o melhor modelo
  3. Validar o modelo com os dados de teste.
    Quando temos um algoritimos que possuem mais de um hiperparmetro como por exemplo Kernel e Gamma em Support Vector Machine e precisamos escolher o melhor conjunto de valores para esses hiperparmetros, fazemos uma tabela com todas as possibilidades (passo 1) para escolher a melhor (passo 2) e validar como os dados de teste (passo 3).

    Como ilustrado na Tabela Abaixo para um algoritimo de Support Vector Machine:

    Kernel/ Gamma Linaer Polimomial
    0.1 0.4 0.6
    1 0.8 0.3
    10 0.5 0.2

Questão 8 - Validação Cruzada¶

O que a tcnica de treinamento de validao-cruzada k-fold? Quais benefcios essa tcnica proporciona para busca em matriz ao otimizar um modelo?
Dica: Assim como h um raciocnio por trs de utilizar um conjunto de teste, o que poderia dar errado ao utilizar busca em matriz sem um conjunto de validao cruzada?

Resposta:
Nunca devemos usar nosso conjunto de teste para treinar o modelo. Entao o ideal termos um conjunto de validao para tomada de deciso em relao ao modelo, com o valor dos parametros por exemplo. Assim nossos dados ficaro divididos em 3 conjuntos: Treinamento, Validao e Teste
A tcnica de validao-cruzada k-fold um mtodo para reciclarmos nossos dados. O objetivo estimar o quo preciso o modelo ser quando novos dados entrarem. Funciona da seguinte forma:

  1. Os dados so quebrados em K subconjuntos mutualmente exclusivos.
  2. Treinamos nossos modelos K vezes, cada vez usando um pedao diferente como conjunto de testes e o restante como conjunto de treinamento.
  3. Tiramos uma mdia dos resultados para escolher o modelo final

    Essa tcnica se ajusta muito bem a utilizao da busca em matrix sem um conjunto de validao cruzada, pois ao dividir os dados em conjunto de treinamento e conjunto de testes apenas uma vez o tamanho dos conjuntos fica limitado e ao usar um nico conjunto de validao pode ocorrer overfiting.

    Assim o grande benefcio de usar a valizao-cruzada k-fold na busca em matriz que todas as observaes so usadas para treinamento e validao, e cada observao usada para validao uma nica vez. Isso nos d a segurana de usar todo o dataset para treinamento e validao sem correr o risco de superestimar o desempenho do conjunto de validao nico que foi selecionado na otmizao do modelo.

    cv_sets = ShuffleSplit(X.shape[0], n_iter = 10, test_size = 0.20, random_state = 0)

    # TODO: Gerar uma rvore de deciso de regresso de objeto
    regressor = DecisionTreeRegressor()

    # TODO: Gerar um dicionrio para o parmetro 'max_depth' com um alcance de 1 a 10
    params ={'max_depth': list(range(1, 11))}

    # TODO: Transformar 'performance_metric' em uma funo de pontuao utilizando 'make_scorer' 
    scoring_fnc = make_scorer(performance_metric)

    # TODO: Gerar o objeto de busca em matriz
    grid = GridSearchCV(estimator=regressor, param_grid=params,scoring=scoring_fnc,cv=cv_sets)

    # Ajustar o objeto de busca em matriz com os dados para calcular o modelo timo
    grid = grid.fit(X, y)

    # Devolver o modelo timo depois de realizar o ajuste dos dados
    return grid.best_estimator_

Fazendo Estimativas¶

Uma vez que o modelo foi treinado em conjunto de dados atribudo, ele agora pode ser utilizado para fazer estimativas em novos conjuntos de entrada de dados. No caso do regressor da rvore de deciso, o modelo aprendeu quais so as melhores perguntas sobre a entrada de dados, e pode responder com uma estimativa para a varivel alvo. Voc pode utilizar essas estimativas para conseguir informaes sobre os dados dos quais o valor da varivel alvo desconhecida por exemplo, os dados dos quais o modelo no foi treinado.
